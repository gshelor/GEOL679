# Function to plot data
def plot_digit(digits, i):
    plt.figure(figsize=(2, 2))
    plt.imshow(digits.images[i], cmap='binary', interpolation='nearest')
    plt.xticks([])
    plt.yticks([])
    plt.text(8, 0.1, str(digits.target[i]), color='green', fontsize=16)

Q4. What have you learned about the effect of training size on accuracy score? Why might it be a bad idea to have a training size near 1.0?¶

Answer: I have learned that training size should be much more than half of the dataset, ideally around 70 or 80% of the dataset, in order to create a model which achieves the highest level of accuracy. However, when training size gets closer to 1.0, the risk of overfitting increases, making the model less usefull for predicting on out-of-sample data.

Q7. What have you learned about n_estimators?¶

Answer: As the number of trees in a random forest model increases, the model generally performs better. However, once the number of trees is large enough, the improvement in performance becomes smaller with each additional tree, and this may make adding more trees not worth it if time or computing power is an issue.
